# M1 里程碑目标：基础命令可用与双输出形态打通

## 1. 里程碑目标
在 M1 完成 `qrgen` 的最小可用闭环：  
- 用户可通过一条命令传入文本内容并生成二维码。  
- 默认情况下在 Terminal 中进行 TUI 渲染。  
- 通过 `-o/--output` 可输出 PNG 文件到指定路径。  

对应 `QRGen.md` 中里程碑建议：  
`M1: 跑通基础命令与默认 TUI 渲染（并支持 -o 输出 PNG）`。

## 2. 范围（In Scope）
- 支持位置参数输入二维码内容（字符串）。
- 支持默认输出：Terminal TUI 渲染。
- 支持文件输出：`-o, --output <FILE>` 输出 PNG。
- 成功输出的二维码可被常见扫码工具识别。
- 基础 CLI 命令可运行（含主流程与最小成功路径）。

## 3. 非范围（Out of Scope）
以下内容不作为 M1 完成条件（后续里程碑处理）：
- 完整参数校验与精细化错误提示（M2）。
- 测试补齐与示例文档完善（M3）。
- SVG/颜色/纠错级别/Logo/批量输入等扩展能力（M4+）。

## 4. 验收标准（M1 DoD）
满足以下条件即可判定 M1 完成：
1. 执行 `cargo run -- "https://example.com"` 时，未传 `-o`，可在终端看到二维码渲染结果。
2. 执行 `cargo run -- "https://example.com" -o ./dist/site.png` 后，目标 PNG 文件成功生成且可被扫码识别。
3. 默认行为与参数行为一致：
   - 不传 `-o` -> Terminal TUI
   - 传 `-o` -> PNG 文件输出

## 5. 交付物
- 可运行的 `qrgen` 命令主流程（输入 -> 生成 -> 输出）。
- Terminal TUI 渲染实现（默认路径）。
- PNG 写文件实现（`-o` 路径）。
- 对应最小使用说明（至少包含上述两条验收命令）。
